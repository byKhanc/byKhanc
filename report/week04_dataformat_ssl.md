# 데이터 교환 포맷 및 시리얼라이징 이해

## 데이터 교환 포맷과 시리얼라이징 개념
- **데이터 교환 포맷**: 서로 다른 시스템이나 애플리케이션 간 데이터를 주고받기 위한 약속된 형식
- **시리얼라이징(Serialization)**: 데이터 구조를 바이트 스트림이나 문자열 형태로 변환하는 과정
- **역직렬화(Deserialization)**: 직렬화된 데이터를 원래 데이터 구조로 복원하는 과정

## JSON, XML, YAML 비교

| 구분      | JSON                                   | XML                                      | YAML                                        |
|-----------|--------------------------------------|-----------------------------------------|---------------------------------------------|
| 개념      | JavaScript 객체 리터럴에서 파생된 경량 데이터 교환 포맷 | SGML에서 파생된 마크업 언어                | 사람이 읽기 쉬운 데이터 직렬화 포맷             |
| 가독성    | 매우 좋음 (간결하고 직관적)            | 보통 (태그가 많아 다소 장황)               | 매우 좋음 (공백과 들여쓰기로 구조 표현)           |
| 구조      | 키-값 쌍과 배열 기반 트리 구조         | 태그와 속성 기반 트리 구조                  | 들여쓰기와 콜론(:) 기반 계층 구조                  |
| 데이터 타입 | 문자열, 숫자, 불리언, 배열, 객체, null 지원 | 모든 데이터를 문자열로 취급 (스키마로 타입 정의) | 문자열, 숫자, 불리언, null, 날짜, 시간 등 다양한 타입 지원 |
| 주요 용도 | 웹 API, 모바일 앱, 비동기 통신, NoSQL | 웹 서비스(SOAP), 문서 마크업, 데이터 저장 및 교환 | 설정 파일, 데이터 직렬화, 로그 파일                  |
| 장점      | - 가볍고 빠름<br>- 파싱 용이<br>- 대부분 프로그래밍 언어 지원 | - 강력한 확장성<br>- 엄격한 유효성 검사 가능<br>- 문서 지향적 | - 뛰어난 가독성<br>- 간결한 문법<br>- 다양한 데이터 타입 지원  |
| 단점      | - 주석 지원 미흡<br>- 복잡한 문서에는 부적합 | - 태그 중복으로 파일 크기 증가<br>- 파싱 복잡 | - 들여쓰기 오류에 민감<br>- 파싱 속도 느림               |

HTTPS, SSL 인증서 기초
HTTPS (Hypertext Transfer Protocol Secure)
HTTP에 보안 기능이 추가된 프로토콜

웹 브라우저와 서버 간 통신 암호화로 데이터의 기밀성, 무결성, 사용자 인증 보장

기밀성(Confidentiality): 데이터가 암호화되어 중간에 가로채더라도 내용을 알 수 없음

무결성(Integrity): 데이터가 변조되지 않았음을 보장

인증(Authentication): 클라이언트가 접속하는 서버가 진짜임을 보장하여 피싱 공격 방지

SSL/TLS 프로토콜을 통해 구현됨

SSL / TLS 인증서
SSL은 TLS의 이전 버전이며, TLS가 현재 주로 사용됨

SSL/TLS 인증서는 서버 신원을 증명하고, 서버-클라이언트 통신을 암호화하는 디지털 파일

공신력 있는 **CA (Certificate Authority)**에서 발급

주요 역할
서버 신원 확인: 방문하는 웹사이트가 합법적이고 신뢰할 수 있음을 증명

데이터 암호화: 사용자 ID, 비밀번호, 결제 정보 등 민감 정보를 보호

작동 방식 (간략)
클라이언트가 HTTPS 사이트 접속 → 서버가 SSL/TLS 인증서 전송

클라이언트가 인증서의 CA 발급 여부 및 유효 기간 확인

성공 시 클라이언트와 서버가 암호화된 세션 키 생성 및 교환

세션 키로 이후 모든 통신 암호화 및 복호화

SSL/TLS 인증서 중요성
사용자 신뢰 확보: 브라우저 주소창 자물쇠 아이콘 및 보안 연결 메시지 표시

데이터 보안 강화: 민감 정보 안전 전송

SEO 유리: HTTPS 사용 사이트에 검색 엔진이 높은 순위 부여

HTTP/2 활성화: HTTP/2는 HTTPS 환경에서만 작동

결론
JSON, XML, YAML은 각각 용도와 특성에 따라 선택하여 사용

HTTPS와 SSL/TLS 인증서는 현대 웹 환경에서 필수적인 사용자 데이터 보호 및 신뢰성 확보 도구
